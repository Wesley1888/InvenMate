# InvenMate - 周转库配件耗材管理系统

一个现代化的桌面库存管理应用，专为周转库配件耗材管理而设计。

## 功能特性

### 🏢 核心功能
- **入库管理** - 完整的入库流程，支持订单关联
- **出库管理** - 出库记录追踪，包含领用人和部门信息
- **订单管理** - 采购订单创建和管理
- **库存管理** - 实时库存状态监控和预警
- **报表统计** - 多维度数据分析和报表导出
- **系统设置** - 灵活的系统配置选项

### 📊 数据管理
- **配件型号统一管理** - 标准化的配件编码和规格
- **差额法计算** - 精确的库存成本核算
- **周转库关联** - 入库出库数据关联追踪
- **库存预警** - 智能库存不足提醒

### 🎯 业务特点
- **订单号管理** - 完整的订单追踪体系
- **数量金额统计** - 详细的财务数据记录
- **发放数量记录** - 精确的出库数量管理
- **结余数量计算** - 实时库存余额显示
- **出库记录追踪** - 完整的领用记录

## 技术栈

- **前端框架**: React 18
- **UI组件库**: Ant Design 5
- **桌面框架**: Electron 27
- **数据库**: SQLite 3
- **构建工具**: React Scripts
- **包管理**: npm

## 系统要求

- Windows 10 或更高版本
- Node.js 16+ 
- 至少 4GB 内存
- 500MB 可用磁盘空间

## 安装和运行

### 1. 克隆项目
```bash
git clone <repository-url>
cd InvenMate
```

### 2. 安装依赖
```bash
npm install
```

### 3. 开发模式运行
```bash
# 启动开发服务器
npm run electron-dev
```

### 4. 生产模式构建
```bash
# 构建应用
npm run dist
```

构建完成后，可执行文件将位于 `dist` 目录中。

## 数据库设计

系统使用SQLite数据库，包含以下主要数据表：

- **part_models** - 配件型号表
- **orders** - 订单表
- **order_items** - 订单明细表
- **stock_in** - 入库记录表
- **stock_out** - 出库记录表
- **inventory** - 库存表
- **departments** - 部门表
- **suppliers** - 供应商表

## 使用说明

### 首次使用
1. 启动应用后，系统会自动创建数据库
2. 进入"系统设置"配置公司基本信息
3. 在"库存管理"中添加配件型号
4. 开始使用入库、出库等功能

### 日常操作流程
1. **入库操作**: 订单管理 → 入库管理
2. **出库操作**: 库存管理 → 出库管理
3. **库存查询**: 库存管理 → 搜索查询
4. **报表查看**: 报表统计 → 选择报表类型

### 数据备份
- 数据库文件位置: `%APPDATA%/InvenMate/invenmate.db`
- 建议定期备份数据库文件
- 可在系统设置中配置自动备份

## 开发说明

### 添加新功能
1. 在 `src/components/` 中创建新组件
2. 在 `src/services/` 中添加相关服务
3. 在 `App.js` 中添加路由
4. 更新数据库结构（如需要）

### 自定义样式
- 全局样式: `src/index.css`
- 组件样式: 使用 Ant Design 主题或内联样式

### 数据库操作
- 使用 `src/services/database.js` 中的方法
- 支持查询、插入、更新、删除操作
